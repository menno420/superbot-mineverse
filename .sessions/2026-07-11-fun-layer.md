## Plan

Final PR of the fun & visuals lane (PR 1 = cave visual theme, merged as
8ee029d). Four deliverables + lane close-out, all inside the standing
constraints (stdlib-only backend, vanilla frontend, no build step, no
contract/schema changes, no audio, reduced-motion aware, keyboard
accessible, honest empty states):

1. **Achievements panel** — `build_achievements(...)` in
   `server/views.py`, a pure derivation over the snapshot, wired into
   `build_views` as a NEW additive `achievements` key (no existing key
   changes shape). Seven deterministic badges with documented
   thresholds: Deep Diver (record_depth == max_depth), Packrat (pack
   total ≥ 200), Coin Magnate (coins ≥ 10 000), Fully Geared (all 9
   schema slots), Tool Breaker (any gear_wear ≥ 100 display cap),
   Balanced Build (≥ 2 skills, spread ≤ 1), The Answer (exactly 42 of
   one pack item). Frontend section renders a roll-up + per-miner badge
   chips (aria-hidden emoji + text labels). Unit tests pin every
   boundary and the sample winners.
2. **Easter eggs** — Konami-code diamond rain (reuses `oreIconSVG`,
   gated by `prefersReducedMotion()`, Escape dismisses, auto-cleans, no
   layout shift); Tool Fondler (tool row becomes a real button, 10
   activations → one-time sparkle + `role="status"` toast); sleeping 💤
   idle state on miner cards when the snapshot is stale (same age math
   as `renderStaleness`, visually-hidden "(idle)" text); a console
   ASCII mine-shaft greeting on boot.
3. **Cave-art 404 page** — `web/404.html` ("you dug too deep"), served
   by a `send_error` override for unknown non-API GET paths only;
   `/api/*` JSON 404/405 + Allow headers stay byte-identical; server
   tests extended.
4. **Miner VS view** — two labelled native `<select>`s, side-by-side
   stat table (depth, record, level, XP, coins, energy, vault level,
   pack total, skill ranks) with aria-hidden delta bars; honest states
   for no pick / same pick.
